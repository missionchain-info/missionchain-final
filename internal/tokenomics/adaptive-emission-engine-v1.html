Section 01

Executive Summary

The MissionChain Adaptive Emission Engine is a self-regulating protocol that dynamically controls MIC token emission based on real-time market conditions. Unlike fixed-schedule emission models (e.g., Bitcoin halving), our engine continuously adjusts emission rate, staking rewards, and MICE license pricing to maintain economic equilibrium.

Design Philosophy One unified mining pool (85% = 5.95B MIC) mined exclusively through MICE licenses. All emission is channeled through MICE mining activity, then split algorithmically into 4 streams. NFTs function purely as staking rights — they do NOT have separate emission pools.

Key Design Principles

1. Activity-Based Emission: MIC is only mined when MICE holders actively contribute (Hindex algorithm). No passive earning.

2. ROI Self-Regulation: The system targets 250% annual ROI for MICE miners. When ROI drifts, emission automatically adjusts to restore balance.

3. Demand-Responsive: More MICE purchased = more emission. Fewer MICE active = emission slows. Natural supply-demand equilibrium.

4. Deflationary Pressure: 5% of all emission is auto-burned, creating permanent deflationary force against inflation.

5. DAO-Governed Evolution: All ratios are adjustable by DAO vote within hard-coded guardrails, allowing the community to adapt as the ecosystem matures.

Section 03

Master Emission Formula

Daily MIC emission is governed by a three-factor multiplicative model that balances time-decay, demand, and ROI stabilization:

E(t) = E_base(t) × D(t) × R(t)
Where:
  E_base(t) = Base emission with exponential decay  // Time-based scarcity
  D(t)      = MICE Demand Factor                 // Market demand signal
  R(t)      = ROI Regulator                     // Profitability stabilizer

// Subject to: cumulative_emitted(t) ≤ 5,950,000,000 (hard cap)
// Subject to: E(t) ≤ 2 × E_base(t) (daily cap)

How the Three Factors Interact

E_base(t) establishes the decreasing baseline — a smooth exponential decay that halves every 180 days. This creates urgency: early MICE buyers get higher base emission.

D(t) scales emission based on actual demand. If MICE utilization is low, emission decreases (no point minting tokens nobody is earning). If utilization is high, emission increases proportionally.

R(t) is the critical self-regulating mechanism. It continuously compares actual miner ROI against the 250% target. If MIC price rises (ROI too high), R(t) drops below 1.0, reducing emission. If MIC price falls (ROI too low), R(t) rises above 1.0, increasing emission. This creates a natural price stabilizer.

E_base(t) = E_0 × 2(-t / T_half)
Where:
  E_0     = Initial daily emission = 22,907,500 MIC/day

Section 05

D(t): MICE Demand Factor

The demand factor ensures emission scales proportionally with actual network participation. No MICE active = minimal emission. Full MICE utilization = maximum emission.

D(t) = 0.5 + U(t)
Where:
  U(t) = N_active(t) / N_max
  N_active(t) = number of active (non-expired) MICE licenses
  N_max = 100,000

Range: D(t) ∈ [0.50, 1.50]
  U = 0.00 (0 MICE)    → D = 0.50  // Half emission
  U = 0.25 (25K MICE) → D = 0.75
  U = 0.50 (50K MICE) → D = 1.00  // Normal emission
  U = 0.75 (75K MICE) → D = 1.25
  U = 1.00 (100K MICE) → D = 1.50  // 50% boost

Why Linear with 0.5 Floor?

The 0.5 floor ensures the system never fully stops emitting even with zero MICE — this maintains a minimum emission for DAO treasury and burn mechanisms. The linear relationship is chosen for simplicity, transparency, and on-chain gas efficiency. The maximum 1.5x boost at full utilization rewards ecosystem growth without creating dangerous emission spikes.

On-Chain Implementation U(t) is computed from the MICE license contract's activeCount() function, which tracks non-expired ERC-1155 tokens via lazy evaluation (checked on every interaction). No off-chain oracle needed.
Section 06

R(t): ROI Regulator

The ROI Regulator is the most critical self-regulation mechanism. It continuously compares the actual annualized ROI of MICE mining against the 250% target and adjusts emission accordingly.

R(t) = clamp( ROI_target / ROI_current(t),  0.5,  2.0 )
Where:
  ROI_target = 2.50                        // 250%
  ROI_current(t) = annual_mic_per_mice(t) × P_mic(t) / P_mice(t)

Breakdown:
  daily_per_mice = E(t) × r_mine / N_active(t)  // Average per MICE
  annual_mic_per_mice = daily_per_mice × 360
  P_mic(t)  = MIC market price (TWAP oracle)
  P_mice(t) = Dynamic MICE price (see Section 8)

Self-Regulation Dynamics

ScenarioROI_currentR(t)Effect
MIC price surges (bull market) 500% 0.50 ↓ Emission halved. Fewer MIC minted, reducing sell pressure
Healthy equilibrium 250% 1.00 No adjustment. System in balance
MIC price drops (bear market) 125% 2.00 ↑ Emission doubled. More MIC minted, increasing mining attractiveness
Extreme bull run 1000% 0.50 (clamped) Max reduction. Prevents emission from going to near-zero

Price Oracle: PancakeSwap V3 TWAP

MIC price is sourced from PancakeSwap V3 Time-Weighted Average Price with a 24-hour window. This prevents flash-loan manipulation. A Chainlink fallback oracle activates if TWAP freshness exceeds 1 hour. The oracle address is updatable via Gnosis Safe 3-of-5 multisig.

Circular Dependency Resolution R(t) depends on E(t), which depends on R(t). Resolution: use R(t-1) (previous day's R value) to compute today's E(t). Day 1 starts with R(0) = 1.0. This creates a one-day lag that smooths oscillations.
Section 07

Emission Split: 60 / 15 / 20 / 5

Every MIC emitted through the mining process is split into four streams. This split happens at the smart contract level — the EmissionController distributes tokens in a single transaction.

E(t) = E_miners(t) + E_dao(t) + E_staking(t) + E_burn(t)

  E_miners(t)  = E(t) × 0.60  // Distributed to MICE miners by Hindex
  E_dao(t)     = E(t) × 0.15  // Sent to DAO Treasury multisig
  E_staking(t) = E(t) × 0.20  // Distributed to NFT stakers
  E_burn(t)    = E(t) × 0.05  // Sent to 0x000...dead (burned)

Stream Details

60% — MICE Miners

Distributed proportionally based on each miner's Hindex score. Higher activity (contributions, quiz completions, task delivery) = higher share of daily emission. This is NOT passive mining — the Hindex algorithm rewards genuine participation.

Miner i's daily reward:
  reward_i(t) = E_miners(t) × Hindex_i(t) / Σ Hindex_j(t)

Hindex(i) = contribution_count × quality_score × recency_factor

15% — DAO Treasury

Automatically sent to the DAO Treasury (Gnosis Safe multisig). Used for ecosystem grants, marketing, partnerships, bug bounties, and operational expenses. Spending requires DAO vote (MFP-NFT holders).

20% — NFT Stakers (Merged Pool)

Single unified staking pool for ALL NFT tiers (MFP-NFT, Platinum, Gold, Silver) plus non-NFT stakers at reduced rate. See Section 9 for detailed staking mechanics.

5% — Auto-Burn

Permanently burned by sending to 0x000000000000000000000000000000000000dEaD. Creates constant deflationary pressure. Over the full emission lifecycle, approximately 297.5M MIC will be burned through this mechanism alone, reducing effective circulating supply.

DAO Governance The emission split ratios are adjustable by DAO vote within these guardrails: r_mine ∈ [50%, 70%], r_dao ∈ [10%, 20%], r_stake ∈ [15%, 25%], r_burn ∈ [3%, 10%]. Maximum change: ±5% per quarter. Sum must always equal 100%.

Dynamic MICE Pricing

MICE license price adjusts dynamically based on utilization rate, creating a natural supply-demand equilibrium. When demand is high (many active MICE), price increases, cooling demand. When demand is low, price drops, stimulating purchases.

P_mice(t) = P_floor + (P_ceil - P_floor) × U(t)

  P_floor = $300 USDT
  P_ceil  = $1,000 USDT
  U(t)   = N_active(t) / N_max

Price Schedule:

Reward Calculation

reward_i(t) = E_staking(t) × W_i / Σ W_j

Where effective weight W_i:
  W_i = min(staked_i, cap_tier) × mult_nft × mult_time

Example: MFP-NFT holder stakes 500K MIC for 180 days
  W = min(500,000, 1,000,000) × 10.0 × 1.50 = 7,500,000

Example: No-NFT holder stakes 30K MIC for 30 days
  W = min(30,000, 50,000) × 0.5 × 1.00 = 15,000
Design Rationale Allowing non-NFT holders to stake (at 0.5×) serves two purposes: (1) broadens the staking base, increasing MIC lock-up and reducing sell pressure, and (2) creates a natural upgrade incentive — a 0.5× staker who buys a Silver NFT gets a 2× effective boost (from 0.5× to 1.0×), driving NFT demand.
Section 10

Staking Ratio Regulator

To maintain healthy liquidity while incentivizing staking, the system dynamically adjusts staking rewards based on the current staking ratio relative to the 40% target.

StakingBoost(t) = clamp( 1 + 0.5 × (S_target - S(t)) / S_target,  0.7,  1.5 )

Where:
  S(t) = totalStaked(t) / circulatingSupply(t)
  S_target = 0.40

Effective staking emission:
  E_staking_effective(t) = E_staking(t) × StakingBoost(t)

Examples:
  S = 20% → Boost = 1 + 0.5 × (0.4 - 0.2) / 0.4 = 1.25  // +25% rewards
  S = 40% → Boost = 1.00                               // On target
  S = 60% → Boost = 1 + 0.5 × (0.4 - 0.6) / 0.4 = 0.75  // -25% rewards
  S = 80% → Boost = 0.70 (clamped)                   // Min boost
Excess Staking Emission When StakingBoost > 1.0, the extra rewards come from an internal buffer accumulated during periods when StakingBoost < 1.0. If the buffer is empty, the excess is redistributed to the DAO treasury. This ensures total emission never exceeds the formula.
Section 11

Circuit Breakers & Safety Systems

Five independent safety mechanisms protect against extreme scenarios:

CB-1: Cumulative Hard Cap

require( totalEmitted + E(t) ≤ 5,950,000,000 )
// Enforced in EmissionController.mint(). Immutable. Cannot be overridden.

When cumulative emission approaches the cap, daily emission is automatically reduced to fit within the remaining allocation. When cap is reached, emission stops permanently.

CB-2: Daily Emission Cap

E(t) ≤ 2 × E_base(t)
// Prevents D(t) × R(t) from creating extreme daily spikes

CB-3: MIC Price Floor Circuit Breaker

if P_mic(t) < $0.001 → emission PAUSED
// Activated when TWAP drops below threshold
// Auto-resumes when P_mic recovers above $0.002 for 24h

CB-4: Unstake Rate Limit

dailyUnstaked(t) ≤ W_max × totalStaked(t)
  W_max = 10% (Admin adjustable via Gnosis Safe 3-of-5)
// Prevents bank runs. Queue-based FIFO when limit reached.

CB-5: Emergency Pause

EmissionController.pause() → requires Gnosis Safe 3-of-5
// Pauses ALL emission, staking, and MICE sales
// Unpause also requires 3-of-5 multisig + 24h timelock
Escalation Protocol CB-1 and CB-2 are automatic (no human intervention). CB-3 is automatic with auto-recovery. CB-4 is admin-adjustable. CB-5 is manual emergency-only. This layered approach ensures the system self-regulates in normal conditions and only requires human intervention in extreme emergencies.

Appendix: Mathematical Proofs

A. Total Emission Convergence

Proof that base emission sums to S_mine:

  ∫0∞ E_0 × 2(-t/T_half) dt
  = E_0 × [-T_half / ln(2) × 2(-t/T_half)]0∞
  = E_0 × T_half / ln(2) × [1 - 0]
  = E_0 × T_half / ln(2)

  Given E_0 = S_mine × ln(2) / T_half:
  = (S_mine × ln(2) / T_half) × T_half / ln(2)
  = S_mine = 5,950,000,000 MIC  ✓

B. ROI Equilibrium Stability

At equilibrium, ROI_current = ROI_target, so R(t) = 1.0.
Perturbation analysis:

  If MIC price increases by δ:
  → ROI_current increases
  → R(t) = ROI_target / ROI_current < 1.0
  → Emission decreases
  → Less MIC supply → supports price (positive feedback)
  → But fewer MIC minted → lower absolute rewards
  → Some miners exit → fewer MICE → D(t) drops
  → Emission further reduces → scarcity increases
  Result: System finds new equilibrium at higher price

  If MIC price decreases by δ:
  → ROI_current decreases
  → R(t) > 1.0
  → Emission increases (more MIC per miner)
  → Higher absolute rewards attract new miners
  → More MICE purchased → D(t) rises
  → MICE revenue increases (buying pressure on MIC via buyback)
  Result: System finds new equilibrium at supported price

C. Auto-Burn Impact Over Lifecycle

Total auto-burned MIC (assuming D × R ≈ 1.0 on average):
  Total_burn = 5% × S_mine = 0.05 × 5,950,000,000
  = 297,500,000 MIC permanently removed

Combined with MICE revenue buyback-and-burn (20% of MICE sales):
  If 100K MICE sold at avg $600 = $60M revenue
  20% buyback = $12M of MIC purchased and burned
  At avg $0.05/MIC = 240M additional MIC burned

  Total deflationary impact: ~537.5M MIC (7.7% of total supply)

D. Emission Split Sum Invariant

Invariant enforced in GovernanceConfig:
  r_mine + r_dao + r_stake + r_burn = 100%

Guardrail bounds:
  min(r_mine) + min(r_dao) + min(r_stake) + min(r_burn)
  = 50% + 10% + 15% + 3% = 78% < 100%  // Feasible

  max(r_mine) + max(r_dao) + max(r_stake) + max(r_burn)
  = 70% + 20% + 25% + 10% = 125% > 100%
// Not all maxes can be reached simultaneously — enforced by sum = 100% constraint
